# Python 异步编程学习笔记

## 💡 我的通俗理解
异步编程就像是一家餐厅的服务员。
- **同步**：一个服务员从头到尾服务一桌客人，点完菜等厨房做好，再上菜，再结账。期间其他客人干等着。
- **异步**：一个服务员同时招呼多桌客人，A桌点菜后去B桌点餐，C桌结账，不用等厨房。厨房好了通知他，他再去上菜。

`await` 就是"我先去忙别的，这件事好了叫我"。

## 📝 核心要素
1. **`async def`**：定义协程函数，调用时返回协程对象，不立即执行
2. **`await`**：等待协程完成，期间让出控制权给其他协程
3. **`asyncio.run()`**：启动事件循环，运行主协程
4. **`asyncio.gather()`**：并发执行多个协程，等待全部完成
5. **`asyncio.create_task()`**：创建后台任务，立即开始执行

### 🤔 优缺点思考
- **好处**：
  - **高并发**：单线程处理大量IO操作，不阻塞
  - **省资源**：协程切换比线程轻量，内存占用小
  - **代码清晰**：用同步的写法写异步逻辑
- **坏处**：
  - **学习曲线**:理解事件循环、协程概念需要时间(异步编程学的头大)
