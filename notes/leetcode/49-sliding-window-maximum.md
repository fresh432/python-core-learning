# LeetCode 239 滑动窗口最大值

## 我的思路（60分钟）
1. 前20分钟：直觉用 max() 遍历窗口，提交后超时
2. 中间25分钟：想到用两个队列——一个维护窗口元素，一个维护最大值。但实现复杂，始终写不出来
3. 看答案后：思路方向对，但可以只用一个单调队列解决。核心洞察是让队列始终保持递减状态，队首永远是当前窗口最大值

## 代码
```python
class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        q = []
        res, n = [], len(nums)
        for i, j in zip(range(1 - k, n + 1 - k), range(n)):
            # i: 窗口左边界, j: 当前元素下标
            if i > 0 and q[0] == nums[i - 1]:
                q.pop(0)  # 队首滑出窗口
            
            while q and q[-1] < nums[j]:
                q.pop()   # 维护递减：新元素更大则弹出队尾小的
            
            q.append(nums[j])
            
            if i >= 0:
                res.append(q[0])  # 队首即最大值
        
        return res
```

## 关键
- 单调递减队列：队首始终是当前窗口最大值
- 队尾维护：新元素入队前，弹出所有比它小的元素（这些元素永无出头之日）
- 队首维护：窗口滑动时，若队首元素刚好滑出窗口，则 pop(0)
- zip(range(1-k, n+1-k), range(n)) 同步遍历窗口左边界和当前元素

## 教训
- 看到"滑动窗口求最值" → 先想单调队列，不是暴力 max()，也不是普通队列
- 单调队列的核心是维护一个"有用"元素的递减序列，而非存所有元素
- 和 209 题（长度最小子数组）都是滑动窗口，但 209 用双指针，239 用单调队列——窗口内需要快速获取最值时才用单调队列
- 双队列思路是过度设计，单队列 + 单调性即可
