# LeetCode 42 接雨水

## 我的思路（60分钟）
1. 前20分钟：写出分层计算的方法（按高度一层层算），但超时
2. 中间20分钟：想到用单调栈，但实现时底部选择不对，始终写不出来
3. 后20分钟：看答案，发现单调栈解法思路多样，最终找到正确版本理解了底部处理

## 代码
```python
class Solution:
    def trap(self, height: List[int]) -> int:
        ans = 0
        s = []
        n = len(height)
        for i in range(n):
            while s and height[s[-1]] < height[i]:
                cur = s.pop()
                if not s:
                    break
                h = min(height[i], height[s[-1]]) - height[cur]
                ans += (i - s[-1] - 1) * h
            s.append(i)
        return ans
```

## 关键
- 单调递减栈：栈中存下标，维护递减高度
- 接水时机：当前高度 height[i] 大于栈顶 height[s[-1]] 时，栈顶 cur 处可以接水
- 底部选择：s[-1]（新的栈顶）和 i（当前柱）作为两边，cur 是凹槽底部
- 高度：min(两边高度) - 底部高度
- 宽度：i - s[-1] - 1（两边之间的距离减1）

## 教训
- 接雨水是单调栈的终极应用，比"下一个更大元素"更复杂，因为需要计算面积
- 之前实现不出来的原因是底部（凹槽）的选择错误：弹出栈顶 cur 后，新的栈顶 s[-1] 才是左边界，i 是右边界
- 和单调栈系列对比：

| 题目              | 单调栈作用 | 计算内容           |
| --------------- | ----- | -------------- |
| 739 每日温度        | 等更大元素 | 距离差            |
| 496/503 下一个更大元素 | 找更大元素 | 值映射            |
| **42 接雨水**      | 找凹槽   | **面积 = 高 × 宽** |

- 分层法 O(n×max_height) 超时，单调栈 O(n) 是标准解
- 看到"柱状图接水/面积" → 先想单调栈
